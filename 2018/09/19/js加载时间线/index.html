    js加载时间线

    首先生成dom树

    css树生成

    在dom树生成后 会生成css树
    dom和cs都生成完毕后会合并生成一样render树
    渲染引擎开始绘制页面
    dom + css = render
    任何一个dom结点 cs结点改变 render都会改变
    render就需要全绘制 这是一个非常影响效率的表现

    就是reflow do节点的增加和删除 节点的宽高变化都会导致重排

    repaint 重绘 局部刷新 只重绘改的地方 比如背景图片 颜色等

    异步加载js

    js 本来是同加载的会阻塞页面的渲染 必须执行完才进行下一步
    因为js是会修改dom 和 css
